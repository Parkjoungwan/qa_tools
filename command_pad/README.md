# Command Pad

## 개요

Command Pad는 자주 사용하는 셸 명령어들을 버튼 형태로 관리하고 실행할 수 있는 간단한 GUI 애플리케이션입니다. 반복적인 커맨드라인 입력을 버튼 클릭 한 번으로 대체하여 작업 효율을 높여줍니다.

## 주요 기능

- **GUI 기반:** `tkinter`를 사용하여 직관적인 그래픽 인터페이스를 제공합니다.
- **동적 명령어 관리:**
    - **추가:** '+' 버튼으로 새 명령어를 추가합니다.
    - **편집:** 버튼 우클릭으로 이름과 명령어를 수정합니다.
    - **삭제:** '-' 버튼 또는 우클릭 메뉴를 통해 불필요한 명령어를 제거합니다.
    - **재정렬:** '⇅' 버튼으로 명령어 순서를 변경합니다.
- **비동기 명령어 실행:**
    - 명령어를 별도 스레드에서 실행하여, 실행 중에도 UI가 멈추지 않습니다.
    - 명령어 실행 중에는 모든 버튼이 비활성화되고 상태 메시지가 표시되어 사용자에게 피드백을 제공합니다.
- **상태 피드백:** 명령어 실행 시작, 완료, 오류 발생 시 하단 상태바에 관련 정보가 표시됩니다.
- **영구 저장:** 모든 변경사항(추가, 수정, 삭제, 재정렬)은 `cmd_pad_new.json` 파일에 자동으로 저장되어 앱 재시작 시에도 유지됩니다.

## 사용법

1.  **애플리케이션 실행**

    ```bash
    python cmd_pad_new.py
    ```

2.  **명령어 관리**
    -   **추가:** '+' 버튼을 클릭하여 이름과 셸 명령어를 입력합니다.
    -   **편집:** 수정할 버튼을 우클릭하고 "편집"을 선택합니다.
    -   **삭제:** '-' 버튼을 누르고 목록에서 선택하거나, 삭제할 버튼을 우클릭하여 "제거"를 선택합니다.
    -   **재정렬:** '⇅' 버튼을 누르고 목록에서 명령어를 선택한 후 "위로 이동" 또는 "아래로 이동" 버튼으로 순서를 조정합니다.

3.  **명령어 실행**
    -   생성된 버튼을 클릭하면 해당 명령어가 실행됩니다.
    -   명령어가 실행되는 동안에는 모든 버튼이 비활성화되며, 완료되면 다시 활성화됩니다.

## 파일 구조

- `cmd_pad_new.py`: 메인 애플리케이션 로직
- `cmd_pad_new.json`: 사용자가 추가한 명령어 목록 저장 파일
